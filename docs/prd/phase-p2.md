## 9. 字段表达式编辑器（FieldExpressionEditor）

业务人员的临时计算需求落点。**严格限定为"档 1：简单字段表达式"**，不做完整 MDX。**P2 仅做"计算度量"模式，"计算字段"延 P3 评估**。

### 入口

- 字段树顶部 "+ 新建计算度量" 按钮
- 数值区右上角 "+ 新建计算度量" 按钮
- 已建字段在字段树底部"我的字段"区显示，可右键编辑/删除

### 编辑器 UI

```
┌────────────────────────────────────────────────┐
│ 字段名称: [利润率                            ]  │
│ 数据格式: [百分比 ▼]                           │
├────────────────────────────────────────────────┤
│ 表达式:                                        │
│ ┌────────────────────────────────────────────┐ │
│ │ ([销售额] - [成本]) / [销售额]              │ │
│ └────────────────────────────────────────────┘ │
│  [插入字段 ▼]  [插入函数 ▼]   [✓ 表达式有效]    │
├────────────────────────────────────────────────┤
│  作用范围: 仅当前视图（保存到 viewConfig）       │
│                                                │
│              [取消]    [确定]                   │
└────────────────────────────────────────────────┘
```

### 支持的语法（最小集）

| 语法 | 示例 | 说明 |
|---|---|---|
| 字段引用 | `[销售额]` | 必须用别名，自动补全 |
| 算术运算 | `+ - * /` | 标准优先级 |
| 括号 | `( )` | 改变优先级 |
| 数字字面量 | `100`、`0.5`、`-1` | |
| 聚合函数 | `SUM() AVG() COUNT() MAX() MIN()` | 包装基础度量 |

### 明确不支持（防 scope creep）

- ❌ 条件表达式（IF / CASE）
- ❌ 字符串函数（CONCAT / SUBSTRING 等）
- ❌ 时间函数（DATEADD / DATEDIFF 等）
- ❌ MDX 的 set 操作 / member 操作 / Filter / Order 等
- ❌ 用户自定义函数
- ❌ 跨字段类型混用（不能 `[销售额] + [产品名]`）
- ❌ "计算字段"模式（行级表达式）—— 延 P3 评估

完整 BNF 和样例见 [appendices.md](appendices.md) 附录 C。

### 校验

- **实时语法校验**：输入时 AST parser 校验，错误高亮 + tooltip 提示
- **字段引用校验**：所有 `[字段]` 必须在 metadata 里存在
- **类型一致性校验**：参与算术的字段必须是数值类型；最终结果必须是数值
- **聚合规则**：表达式整体最终是个聚合后的标量；可以用 `SUM([销售额]) - SUM([成本])` 显式聚合，或直接 `[销售额] - [成本]`（自动按字段在 metadata 中的默认聚合包装）

### 翻译到后端

- 内部 AST → MDX 表达式 → `customElements.CustomCalcMeasure.measure.expr`
- 简单算术映射：`[A] - [B]` → `[Measures].[A] - [Measures].[B]`
- 聚合映射：`SUM([A])` → `Sum({...}, [Measures].[A])`
- 翻译模块**必须有充分单测**（每种语法节点至少 3 个 case）

### 保存范围

- 仅保存在 `viewConfig.customFields` 数组里
- 跟视图走，分享视图时一起分享
- 不写回 Smartbi metadata
- 想"永久化"，请回 Smartbi 数据集编辑器

### ViewConfig 中的形态

```json
{
  "customFields": [
    {
      "id": "uf_1234",
      "name": "利润率",
      "kind": "calc_measure",
      "dataFormat": "百分比-保留一位小数",
      "expression": "([销售额] - [成本]) / [销售额]",
      "ast": { /* 解析后的 AST，用于翻译和回显 */ }
    }
  ]
}
```

### P2 不做（明确边界）

- 表达式 lint / 性能提示
- 表达式版本历史
- 字段间循环引用检测（保留为 P3）
- 自建字段也作为另一个自建字段的输入（可链式引用）—— 暂不支持，必须基于原始字段
- "计算字段"模式

### P3+ 可能扩展

- 条件表达式（IF）
- 自建字段相互引用
- 表达式从其他视图复制粘贴
- "计算字段"模式（如评估通过）

## 10. 维度分组编辑器（DimensionGroupingEditor）

> **使用预定义分组（已在 P0 支持）**：metadata 里 `CALC_GROUP` 字段在 P0 直接显示在字段树并可拖拽使用。
>
> **本节专指 P2 引入的"用户自建分组"。**

两种子类型：枚举分组 + 范围分组。**两者 UI、ViewConfig schema、后端翻译完全独立**，工程实现时不要试图复用同一套抽象。

### 两种分组本质区别

| 维度 | 枚举分组 | 范围分组 |
|---|---|---|
| 基准字段类型 | 离散维度（STRING） | 连续数值（INTEGER/DOUBLE） |
| 用户操作 | 拖拽成员到组 | 输入区间边界 |
| 分组完整性 | 可能有未归类成员（需 `ungroupedHandling` 处理） | 必须连续覆盖（无"未归类"概念） |
| 后端 ColumnDef | `EnumGroupColumn` | `RangeGroupColumn` |
| ViewConfig kind | `enum_group` | `range_group` |
| ViewConfig 主字段 | `groups: [{label, members[]}]` + `ungroupedHandling` | `ranges: [{min, max, label}]` |

### 10.1 枚举分组（按成员归类）

#### 入口

- 任意维度字段右键 → "新建分组"
- 字段树顶部 "+ 新建分组" → 选择基准维度

│ │ □ ...    │    │  ├──────────────────────────┤ │
│ │          │    │  │ 长三角               ✏️ │ │

#### 关键交互

- 加载维度成员（懒加载，支持搜索/分页）— 依赖[阻塞项 10](1-product.md) 的成员加载接口
- 多选成员后批量拖入某组
- 单个成员拖拽到组之间移动
- 组重命名 / 删除
- "未分组成员"两种处理：各自显示 vs 归为"其他"组

#### 翻译到后端

- → `customElements.CustomColumn` + `column.define: { _enum: 'EnumGroupColumn', ... }`
- **完整 schema 待和后端确认**（见 [1-product.md](1-product.md) 阻塞项 8）

### 10.2 范围分组（按数值区间）

#### 入口

- 任意数值字段右键 → "新建范围分组"
- 字段树顶部 "+ 新建分组" → 选择"范围分组"

#### 编辑器 UI

```
┌──────────────────────────────────────────────────┐
│ 基准字段: 年龄                                    │
│ 新分组名称: 年龄段                                │
├──────────────────────────────────────────────────┤
│ 区间设置:                                        │
│   ┌────┐  ≤ 年龄 < ┌────┐  → ┌──────────┐  ✕    │
│   │ 0  │           │ 18 │     │ 未成年    │       │
│   └────┘           └────┘     └──────────┘       │
│   ┌────┐  ≤ 年龄 < ┌────┐  → ┌──────────┐  ✕    │
│   │ 18 │           │ 60 │     │ 青壮年    │       │
│   └────┘           └────┘     └──────────┘       │
│   [+ 添加区间]                                    │
│                                                  │
│   ⚠️ 区间不能重叠，必须按升序                       │
│   边界值：左闭右开 [a, b)                          │
│                                                  │
│ 作用范围: 仅当前视图                              │
│              [取消]      [确定]                   │
└──────────────────────────────────────────────────┘
```

#### 关键校验

- 区间不能重叠
- 必须按升序排列（自动排序或拒绝保存）
- 第一个区间下界默认 `-∞`，最后一个上界默认 `+∞`
- 标签名不能重复
- 至少 2 个区间才能保存

#### 翻译到后端

- → `customElements.CustomColumn` + `column.define: { _enum: 'RangeGroupColumn', ... }`

### P2 分组不做（明确边界）

- 嵌套分组（分组的分组）
- 分组依赖另一个自建分组
- 动态分组（按规则自动归类，如"销售额 top 20%"）
- 分组导出 / 模板

### P3+ 可能扩展

- 动态分组（top-N、按某度量分位）
- 分组保存为模板复用
